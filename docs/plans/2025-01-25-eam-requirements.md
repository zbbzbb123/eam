# EAM 投资决策系统 - 需求文档

> 文档版本: 1.0
> 创建日期: 2025-01-25

---

## 1. 项目背景

### 1.1 用户画像
- **身份**: 开发者，非专业投资人
- **目标**: 让个人理财不再盲目，建立系统化的投资决策流程
- **时间预算**: 每周 1-2 小时用于投资决策
- **技术能力**: 可以自己开发和维护系统

### 1.2 当前痛点
- 完全凭感觉买卖，没有固定流程
- 不知道什么时候该买、什么时候该卖
- 信息太杂，不知道哪些重要，容易被情绪带着走
- 缺乏全局观，不清楚整体仓位的风险敞口和配置是否合理
- 对仓位控制、风险控制、如何对冲风险都很盲目

### 1.3 投资框架 (来自老板)
三层仓位配置：
- **稳健层 (40%)**: VOO、SCHD、GLD、SGOV、XLV 等大指数ETF
- **中等风险层 (30%)**: QQQ、半导体ETF、新能源ETF
- **Gamble层 (30%)**: 个股如晶泰科技、阿里巴巴、白银ETF等，每周review

### 1.4 投资市场
- 美股
- A股
- 港股

---

## 2. 系统定位

**决策辅助系统，不是自动交易系统**

- 系统负责: 采集信息、分析数据、发现机会、给出建议
- 用户负责: 最终决策、执行交易

---

## 3. 核心需求

### 3.1 仓位管理
- [ ] 记录每笔交易（买入/卖出）
- [ ] 强制填写买入理由（便于复盘）
- [ ] 设置止盈止损价格
- [ ] 自动计算三层仓位配比
- [ ] 偏离目标配比时提示再平衡

### 3.2 信息输出方式
- [ ] **每周报告**: 固定时间收到一份汇总，一次性review
- [ ] **实时推送**: 有重要信号立即通知（Telegram）
- [ ] **Web界面**: 查看仓位、信号、历史记录

### 3.3 信息输入范围
- [ ] 宏观经济: 经济周期、利率、通胀、地缘政治
- [ ] 个股数据: 财报、估值、技术指标、公司新闻
- [ ] 市场情绪: 资金流向、期权数据、社交媒体热度

### 3.4 决策支持
- [ ] **选什么**: 每层应该配哪些具体标的
- [ ] **何时买**: 什么价位/时机入场
- [ ] **何时卖**: 什么时候该止盈或止损
- [ ] **再平衡**: 仓位偏离后如何调整

## 5. 板块分析需求

### 5.1 板块一：硬科技与新要素

#### AI & 算力
- 监测指标:
  - Mag 7 巨头资本开支 (Capex)
  - 开发者热度 (GitHub Star、HuggingFace下载量)
  - arXiv 新论文/新架构
- 关联ETF: QQQ, SOXX, SMH, 515070(A股AI ETF)

#### AI + 生物医疗
- 监测指标:
  - FDA 审批进度
  - ClinicalTrials.gov 临床试验状态
  - 一级市场融资热度 (Crunchbase/36Kr)
- 关联ETF: XBI, IBB, 恒生医疗ETF

#### 新能源
- 监测指标:
  - 上游价格 (碳酸锂、多晶硅)
  - "新三样" 出口数据
  - 产能出清信号
- 关联ETF: 光伏ETF(515790), 新能源车ETF(515030)

### 5.2 板块二：硬通货与宏观对冲

#### 黄金 & 白银
- 监测指标:
  - 美国实际利率 (TIPS Yield) - 核心驱动
  - 金银比 (>85 白银低估, <65 白银高估)
  - 央行购金数据 (世界黄金协会)
- 关联ETF: GLD, IAU, SLV, 黄金ETF(518880)

### 5.3 板块三：地缘政治与宏观环境

- 监测内容:
  - FOMC 声明 Diff 分析 (鹰/鸽变化)
  - 地缘紧张度指数 (关键词监测)
  - 美国宏观数据 (CPI/GDP/失业率/PMI)
  - 中国宏观数据 (PMI/社融/M2)
- 数据源: FRED, GDELT Project

## 6. 个股分析需求

### 6.1 策略一：聪明钱追踪

**美股:**
- 13F 机构持仓分析 (ARK, 桥水, 高瓴等)
- 内部交易监控 (CEO/CFO 大额买入)

**A股/港股:**
- 北向资金持仓变化
- 公司回购 (尤其注销式回购)

### 6.2 策略二：产业链挖掘 (铲子股)

- 从龙头公司年报/研报中提取供应链关系
- 构建产业链图谱 (Neo4j)
- 当龙头大涨时，扫描估值未起飞的供应商
- 供应链风险传导预警

### 6.3 策略三：硬核指标 (另类数据)

- AI 领域: GitHub 项目热度、HuggingFace 模型下载量
- 生物医药: 临床管线状态变化 (Phase升级 = 催化剂)

### 6.4 策略四：基本面排雷

自动化财务打分:
- R&D 占比 (硬科技必看)
- PEG (成长股估值)
- 自由现金流 (重资产行业)
- 资产负债率
- ROE 稳定性

## 7. 持仓跟踪需求

### 7.1 持仓记录
- 股票代码、市场、仓位层
- 持有数量、平均成本
- 买入日期、买入理由
- 止损价、止盈价
- 自定义监控关键词

### 7.2 持仓专属监控
对每只持仓股自动配置:
- 价格告警 (止损/止盈/单日大幅波动)
- 基本面变化 (财报日期、估值分位)
- 相关新闻/公告
- 聪明钱动向 (机构/内部人操作)
- 产业链传导 (上下游重大变化)

### 7.3 AI 决策建议
针对每只持仓生成:
- 当前状态评估 (买入理由是否仍成立)
- 建议操作 (持有/加仓/减仓/清仓)
- 关键关注点 (未来需要盯住什么)
- 下一个催化剂事件

## 8. 输出需求

### 8.1 周报内容
1. 仓位全景: 三层配比、偏离度、再平衡建议
2. 板块信号汇总: 各板块本周关键信号
3. 机会发现: 聪明钱/铲子股/硬核指标发现的机会
4. 风险预警: 持仓风险、宏观风险
5. 本周待办: 需要做决策的事项清单

### 8.2 实时告警

**价格类:**
- 单日涨跌幅 > 5%
- 触及止损/止盈线
- 创52周新高/新低

**事件类:**
- 持仓股财报发布
- FDA审批结果
- 高管大额买入
- 机构持仓变化

**宏观类:**
- FOMC会议结果
- 重要经济数据发布
- 地缘紧张度指数飙升

### 8.3 Dashboard
- 仓位可视化 (三层配比图)
- 持仓明细 (盈亏状态)
- 信号时间线
- 宏观仪表盘

## 9. 技术需求

### 9.1 技术栈
- 语言: Python 全栈
- 后端: FastAPI
- 前端: Vue 3 + ECharts
- 数据库: MySQL + Neo4j
- 部署: Docker Compose
- 运行环境: 云服务器 (24小时在线)

### 9.2 非功能需求
- 数据采集稳定，爬虫失败有告警
- 每周定时生成周报
- 紧急信号秒级推送
- 所有建议可追溯到原始数据

---

## 10. 约束与假设

### 10.1 约束
- 用户每周只有 1-2 小时处理投资事务
- 系统必须高度自动化，只推送关键信息
- 不做自动交易，只做决策建议

### 10.2 假设
- 用户有基本的投资知识，能理解系统输出
- 用户能自己执行交易（券商APP）
- 数据源 API 保持可用
